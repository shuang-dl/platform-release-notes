ACT-877 Merged Batch Import: pre-flight duplicate email & phone validation Now validates duplicate tenant email and mobile phone before import, so conflicts are caught early with accurate row errors. Previously, duplicate tenant email or mobile phone errors only surfaced after the batch import job had already run — causing the entire job to appear as failed. Now a validation step runs before the job starts, surfacing duplicate-email and duplicate-phone conflicts as clear row-level error messages upfront, so PMs can fix issues before any data is processed. 🟢 MATCHED

ACT-849 Merged Post-migration Slack alerts: add Implementation & Onboarding Owner Added Implementation and Onboarding Owner names to post-migration Slack alerts, with N/A fallback when unavailable. The internal #data-migration-updates Slack alerts that fire when a tenant's data migration completes now include the Implementation Owner and Onboarding Owner names (pulled from the Salesforce account). ACT-928 Merged Salesforce: skip redundant Onboarding writes for fully-onboarded tenants Now skips redundant Salesforce onboarding updates for fully onboarded tenants when nothing has changed, reducing API usage. The Salesforce sync was unconditionally writing one Onboarding__c record per tenant per sweep — even for fully-onboarded tenants with no changed data. A new one-shot short-circuit skips the write when all onboarding milestone timestamps are already set and the score hasn't changed, cutting ~22k–23k unnecessary Salesforce API calls per day. ⚫ INTERNAL — Backend performance (NFR) optimization; no customer-facing behavior change. — no related video
