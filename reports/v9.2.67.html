ACT-398 Batch import: retry recurring-payment creation on Mongo write conflicts Now retries recurring payment creation during batch imports when database write conflicts occur, improving import reliability. BATCH_IMPORT jobs were hitting MongoDB Write conflict during plan execution errors (~23/week) when downstream syncs (Intercom user sync, Stripe payout acceleration) collided on the same lease/tenant/recurring-payment documents. The fix wraps the affected downstream paths with the existing WRITE_CONFLICT-aware retry helper so imports succeed instead of failing mid-flight. INTERNAL RELIABILITY FIX — no Help Center change

No user-facing UX change. Imports that previously hit transient conflicts will now succeed without manual rerun.
ACT-649 2FA: clearer "remember this device for 30 days" checkbox copy Added clearer 2FA checkbox text so users know it skips the verification code on this device for 30 days. The 30-day checkbox on the 2FA verification screen previously confused users — some thought it skipped login entirely, some didn't know what 2FA was, some didn't understand the browser-session implication. Copy is now explicit: this device will skip the verification code for 30 days. Goal is to cut support tickets driven by misunderstanding the option.
