Summary

PMO-2112 AI SMS assistant now remembers the full conversation Added a running conversation summary and now uses all messages since it for ongoing SMS replies. The SMS auto-responder previously dropped context if there was a gap of a week or more between tenant messages. It now keeps a single running summary and includes every message sent since that summary when generating replies, so the AI has continuous context instead of losing the thread on longer-running conversations. TEN-797 Payment hold infrastructure for risk review (backend) Added held-payment records for risk review and a bypass for payment-limit revalidation during approval. Backend-only groundwork for the upcoming Payment Hold & Risk Review feature. Adds a lightweight HeldPayment side record (instead of a pending journal entry) so a flagged payment doesn't touch the ledger until it's approved or declined, plus a bypassLimitValidation flag so an approved payment isn't re-rejected by the payment-limit check. No UI or tenant/PM-facing workflow ships in this ticket — it's the data model and repository logic that a future approval flow will build on. BETA — building block for the in-progress "Payment Hold & Risk Review" project (milestone: "[M6] Build"); hold for GA before documenting. Related existing articles for context once GA'd: 'Pending Review' Status on Merchant Account Transactions, Tenant Payment Limit Errors. — no related video

L2L-1654 Custom fields now visible directly in Unit Overview Added a collapsible Custom fields section to Unit Overview with search, add, and edit actions. The redesigned Unit drawer's Overview tab now has a collapsible "Custom Fields" card, mirroring the one already on the Prospect drawer. It shows the DbTenant's custom fields configured for units, with a search box in the header plus Add Fields and Edit Fields actions, label/value rows in the body, and an empty state when none are configured — so PMs no longer have to leave the Unit drawer to check or edit a unit's custom field values.
