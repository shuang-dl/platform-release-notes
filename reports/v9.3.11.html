Summary

This release ships 7 changes across PM Operations, Onboarding & Imports, Tenant Experience & Payments, and Banking. The standout customer-visible change is ACT-619, which ensures the correct official import templates (Residential, Commercial, HOA) are served in GSF v2 — directly affecting the "Import Your Data Into DoorLoop" Help Center article. The remaining customer-visible items are a work-order requesting-tenant picker (behind a feature flag, not yet live) and a fix for owner portal statement report errors; the Tenant Experience and Banking changes are all backend infrastructure improvements with no direct Help Center impact.

TEN-493 Notification Settings API (Infrastructure) Added tenant notification settings retrieval so tenants can view and manage lease-based preferences. Migrates the GET /api/tenant-portal/notification-settings endpoint from the legacy app server to the new decoupled tenant-portal-server. Includes lease settings and late-fees logic. The tenant-facing behavior is unchanged; the endpoint is only routed to the new server when the TenantPortalUseNewServer feature flag is on. This is a backend infrastructure migration with no visible UI change. ⚫ INTERNAL — backend API migration; no Help Center change needed — no related video
TEN-528 Feature Flag Evaluation + Redis Cache (Infrastructure) Added tenant-aware feature flag evaluation with Redis caching for faster, more consistent flag behavior. Wires up the OpenFeature provider and a Redis-backed flag-definition cache in the tenant-portal-server. This enables server-side feature flag evaluation (using PostHog + Redis) for tenant-portal endpoints, with per-request evaluation context built from the tenant's identity. Re-enables per-event feature flag enrichment in PostHog analytics. No customer-facing behavior change; purely infrastructure. ⚫ INTERNAL — NFR infrastructure; no Help Center change needed — no related video
TEN-598 Onboarding State APIs (Infrastructure) Added onboarding status updates that save tenant progress more reliably during Short Onboarding. Migrates the tenant onboarding state GET and PUT endpoints from the legacy app server to the new tenant-portal-server. This makes the Short Onboarding flow more reliable by routing through the decoupled server. No new UI or behavior from the tenant's perspective; purely an infrastructure migration as part of the Decouple Tenant Portal Server project. ⚫ INTERNAL — NFR infrastructure; no Help Center change needed — no related video

Banking 1 item

Linear Feature Original Notes (Slack) What it does Affected articles Affected videos
BAN-947 Full Payout Failure Chain — Deposit Line Items Fix Now includes original line items from the full Stripe payout failure chain, keeping deposit history complete. A bug caused DoorLoop to silently drop original charge line items from a successful payout deposit when a Stripe payout had failed more than once before succeeding. The fix walks the entire failure chain iteratively (not just one level deep), so all original line items are captured in the deposit record. Also fixes the same blind spot in the manual force-deposit intervention tool. Affects accounting accuracy for merchants with repeated payout failures. High-priority internal accounting fix; no UI change. ⚫ INTERNAL — backend Stripe accounting fix; no customer-facing article needed — no related video
