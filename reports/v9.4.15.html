Summary

This release ships three Tenant Experience & Payments changes — a new tenant-portal insurance-required dialog, PM-facing "Request policy" alerts on lease overview pages, and a backend fix scoping the payment-hold auto-decline sweep to eligible companies — plus one Banking fix so consent-check calls no longer 401 prospects out of the public rental-application wizard.

TEN-966 Payment-hold auto-decline sweep scoped to eligible companies Fixed payment-hold auto-decline to run only for eligible companies and exit cleanly when none are enrolled. Backend fix: the hourly payment-hold auto-decline job was running an unindexed cross-tenant scan of a 52.5M-row collection every run. The discovery query is now scoped to companies actually enrolled in payment-hold review and short-circuits entirely when none are enrolled — a database/performance fix with no visible UI change. ⚫ INTERNAL — backend performance fix only; no user-facing behavior or UI changed, no Help Center update needed. — no related video

Linear Feature Original Notes (Slack) What it does Affected articles Affected videos
BAN-1409 Rental-application wizard no longer breaks on consent check Fixed consent prompts so they only appear for authenticated property managers and no longer break applicant rental applications. A PM-only banking "get-consent" check was accidentally firing inside the public rental-application wizard, returning a 401 that threw applicants out with a "Something went wrong" error. The check is now gated to PM identities only, so prospects can complete their rental application without interruption. ⚫ INTERNAL — bug fix restoring existing rental-application flow; no new feature or setting to document.
