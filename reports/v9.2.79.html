Summary

This release ships 7 changes across PM Operations (3), Onboarding & Imports (3), and Tenant Experience & Payments (1). The customer-visible highlights are: vendor inactive status with toggles and list filtering (PMO-986), portfolio filtering in Communications Center (PMO-1462), enhanced Communications Center UI with tenant presence indicators and auto-responder controls (PMO-641), and localized SSO invitation error messages for the tenant portal (ACT-612). Three items are internal infrastructure with no Help Center impact.

PMO-986 Vendor inactive/active status Added an inactive status for vendors with active/inactive toggles, list filtering, and a guard for recurring transactions. Property managers can now mark vendors as inactive (or re-activate them) via a toggle on the vendor overview page. The vendor list gains an active/inactive filter (default: active only, mirroring property behavior). Recurring transactions that reference an inactive vendor are guarded to prevent unintended charges. MATCHED

  • Add a Vendor — Note that newly-added vendors are active by default; show how to deactivate after creation.
  • Edit a Vendor — Add inactive toggle to the "what you can edit" list and update screenshots.
 — no related video

A new portfolio (property group) filter is available in the Communications Center inbox, letting users narrow messages and conversations to a specific portfolio. Works the same way as property-level filtering already present in the UI. MATCHED

Onboarding & Imports 3 items

Linear Feature Original Notes (Slack) What it does Affected articles Affected videos
ACT-719 Batch import error readability (internal) Fixed batch import failures to save a readable error message, making failed jobs easier to review and clean up. Internal improvement: batch import job failure reasons are now stored as a human-readable string (e.g. "Property not found") instead of a raw JSON object in the database. This makes failed-job logs easier for ops/support to query and clean up. No change to the user-facing import UI. ⚫ INTERNAL — Backend log format change only; the property manager's import screen is unchanged. Labels: NFR, Maintainability. — no related video
ACT-738 2FA analytics to New Relic (internal) Added 2FA verification tracking with outcome, reason, and channel details for better login funnel reporting. Internal monitoring: the OTP/login transaction now emits custom New Relic attributes (otpOutcome, otpReason, otpChannel) so the 2FA dashboard can show a real per-reason success funnel. No change to user-facing 2FA flows. ⚫ INTERNAL — Pure analytics/observability change. No customer-visible behavior. Label: NFR. — no related video
ACT-612 Localized SSO invitation error messages Now shows localized SSO invitation and login error messages instead of raw codes for expired or invalid access. Tenants who click an expired or invalid SSO invitation link now see a clear, localized message in English or Spanish (e.g. "This invitation link has expired. Please ask your property manager to send a new one.") instead of the raw code string invitation_expired. Four error codes are covered: invitation_expired, invitation_invalid, no_portal_access, db_tenant_not_found. MATCHED
  • How to Accept Your Tenant Portal Invite (Step-by-Step) — Add a troubleshooting step showing the new "link expired" and "link invalid" error messages tenants may see.
  • Troubleshoot Tenant Portal Login Issues — Add an SSO error codes section documenting all 4 new messages and what tenants should do for each.
  • Fix Common Tenant Portal Invite Issues — Add "Expired or invalid invite link" scenario with the new message copy and resolution (re-invite from DoorLoop).
 VIDEO MATCHED
