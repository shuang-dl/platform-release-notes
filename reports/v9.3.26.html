Summary

This is a small release with two changes: an Onboarding & Imports polish item that adds an animated saving/saved/error indicator to the Get Started flow (ACT-1019), and a Banking backend change that adds background syncing for Stripe Treasury financial accounts and transactions (BAN-1088). Only the Get Started indicator is customer-visible enough to touch existing Help Center content; the Treasury sync work is internal data-pipeline infrastructure with no new UI of its own.

ACT-1019 Animated save-status indicator in Get Started Added an animated save-status indicator in Get Started that shows saving, saved, and error states. The static green dot in the Get Started bottom bar's "Progress saves automatically" pill is replaced with an animated icon that reacts to the real save state: a spinner draws in while saving, then a checkmark draws in and rests once the save succeeds, or an error affordance shows if the save fails. It respects prefers-reduced-motion and announces state changes to screen readers. MATCHED

Banking 1 item

Linear Feature Original Notes (Slack) What it does Affected articles Affected videos
BAN-1088 Background sync for Stripe Treasury accounts Added background syncing for Stripe Treasury financial accounts and transactions, with automatic webhook-triggered updates. Adds two new "raw mirror" MongoDB collections and matching repositories that replicate Stripe Treasury FinancialAccount and Transaction objects, kept in sync via webhook events (treasury.financial_account.*, and the various outbound/inbound transfer and payment events) plus BullMQ backstop jobs. This is decoupled from the existing Embedded Financing domain logic and ships with no feature flag. INTERNAL — Purely backend data-sync infrastructure (new DB collections, repositories, webhook handler, BullMQ jobs); no new screen, setting, or workflow for property managers to learn, so no Help Center change is needed. — no related video
