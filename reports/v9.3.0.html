Summary

Release v9.3.0 — DoorLoop's first major version bump — ships 16 changes across four product areas. The headline customer-facing features are lease document tracking with tenant portal sharing (PMO-203), automatic SMS/voice fallback for international 2FA delivery (ACT-744), Lemonade renters insurance expansion to five new states (TEN-579), and a redesigned Pay-or-Quit workflow template with a distinct lavender look and first-in-list placement (WOR-1123). Owner Portal file search and filtering (PMO-1783) and the Late Balance Notice automated workflow execution (WOR-430) ship behind feature flags. Six items are internal infrastructure or analytics with no Help Center impact.

ACT-744 International 2FA — Automatic SMS/Voice Fallback Now supports automatic SMS or voice fallback for international 2FA codes when one delivery channel is blocked. When a 2FA verification code can't be delivered via SMS (e.g., alphanumeric sender ID "DoorLoop" blocked by a country's carrier, such as Pakistan), the system now automatically retries via voice call — and vice versa. Previously both channels could fail silently, showing a misleading "check or change your number" error even when the number was valid. The fix also adds the missing countries to Telnyx's outbound voice profile whitelist. Users in blocked regions (Pakistan and others) will now reliably receive their OTP. MATCHED
  • Troubleshoot 2FA Verification Code Issues — Add a section on international SMS delivery: explain that DoorLoop now automatically falls back to voice if SMS is blocked; list countries where alphanumeric sender IDs may not work; update the "change your number" guidance to distinguish a carrier-blocked scenario (change nothing) from a genuinely invalid number.
  • 2FA Verification Code — Add a note that users in some countries may receive their OTP via voice call instead of SMS; explain the automatic fallback.

ACT-812 Data Migration — Custom Lease Names Preserved on Import Now supports preserving custom lease names during import instead of auto-renaming them from tenant data. The data migration sheet now accepts an optional lease.name column. When populated, that value becomes the lease name in DoorLoop instead of the default auto-generated name (concatenated from tenant first/last name fields). Designed for upmarket accounts that pre-define lease names aligned to their own naming conventions (e.g., unit codes, owner entity names). If the field is blank, the existing auto-naming behavior is unchanged. MATCHED
