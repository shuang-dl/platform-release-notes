# Platform Release Notes

A live, internal viewer for DoorLoop platform release impact reports — generated by Claude and deployed via DeployBay.

## What it does

Every time a new platform release ships, Claude reads the release post from the `#platform-release-notes` Slack channel, cross-references every feature against the DoorLoop Help Center and Training Hub, and generates an HTML impact report. That report gets committed here automatically, and this site surfaces them all in one place.

Each report shows:
- Which Help Center articles are likely affected and what needs updating
- Which Training Hub videos are likely affected and whether they need re-recording
- Features with no existing coverage (candidates for new articles or videos)

## How a new report gets added

1. In Claude Code, run: `release summary for v9.2.XX` (or `summarize the latest release`)
2. Claude generates the report and automatically pushes it to this repo
3. Redeploy in DeployBay — the new report appears in the table

No manual file management needed.

## Repo structure

```
index.html        # Main viewer page — loads reports.json and renders the table
reports.json      # Manifest of all reports (date, version, title, filename)
reports/          # One HTML file per release (e.g. v9.2.55.html)
Dockerfile        # nginx:alpine — serves everything as static files
```

## Deploying

This is a static site served by nginx. The Dockerfile is self-contained — no build step, no env vars needed.

**DeployBay settings:**
- Repo: `shuang-dl/platform-release-notes`
- Port: `80`
- No environment variables required

## Adding a report manually

If you need to add a report outside of Claude:

1. Drop the HTML file into `reports/`
2. Add an entry to `reports.json`:
   ```json
   {
     "date": "YYYY-MM-DD",
     "version": "v9.2.XX",
     "title": "Release v9.2.XX — Help Center Impact",
     "filename": "reports/v9.2.XX.html",
     "generatedAt": "YYYY-MM-DDT00:00:00Z"
   }
   ```
3. Push and redeploy
